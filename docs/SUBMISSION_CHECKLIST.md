# OpenAI Build Week submission checklist

Official deadline: **July 21, 2026 at 5:00 PM PDT** — **July 22 at 9:00 AM JST**.

## Eligibility and registration

- [ ] Join the challenge on https://openai.devpost.com/
- [ ] Confirm the entrant is above the age of majority and eligible in Japan
- [ ] Select **Apps for your life**
- [ ] Read the official rules on Devpost

## Product

- [x] Working PWA
- [x] Immediate sample experience
- [x] Search, theme filtering, edit, delete, export, and import
- [x] Offline and iPhone home-screen support
- [ ] Confirm the final demo URL is accessible to judges without the owner's private access
- [ ] Test the final URL on a real iPhone after the last deployment
- [ ] Test with VoiceOver and dark mode once on the final build

## Repository

- [x] Setup and test instructions
- [x] Architecture and privacy explanation
- [x] Sample-data judging flow
- [x] MIT license
- [ ] Commit the current SwiftUI, PWA, docs, and license files
- [ ] Push the final repository to GitHub
- [ ] Make the repository public, or invite `testing@devpost.com` and `build-week-event@openai.com`
- [ ] Confirm a fresh clone builds successfully

## Demo video

- [ ] Record using [`DEMO_SCRIPT.md`](DEMO_SCRIPT.md)
- [ ] Keep it under three minutes
- [ ] Include audio explaining how Codex **and verified GPT-5.6** were used
- [ ] Upload as a public YouTube video
- [ ] Add captions
- [ ] Put the video URL in Devpost and [`SUBMISSION.md`](SUBMISSION.md)

## Codex evidence

- [ ] Run `/feedback` in the Codex session containing the majority of the core work
- [ ] Copy the session ID into Devpost and [`SUBMISSION.md`](SUBMISSION.md)
- [ ] Verify the session/model was GPT-5.6 before using that wording
- [ ] Highlight key product decisions, not only generated code

- [ ] Review the English for personal voice
