# Build Week demo video script

Target length: **2:30–2:45**. Devpost requires a public YouTube video under three minutes with audio explaining both the product and how Codex and GPT-5.6 were used.

## Recording setup

- Record an iPhone screen in portrait orientation.
- Turn on Do Not Disturb and remove personal notifications.
- Start with a fresh Safari profile or cleared imakore data.
- Use the deployed URL, not localhost.
- Record voice separately only if that makes the narration calmer and clearer.

## Script and shots

### 0:00–0:15 — Problem

**Shot:** imakore's empty “振り返る” screen.

**Narration:**

> When your mind is already full, most note apps ask you to do more work: choose a folder, a tag, a priority, or a deadline. imakore is for people who want to write first and organize nothing.

### 0:15–0:40 — Capture

**Shot:** switch to “残す,” enter「写真を撮る時間を、もう少し大切にしたい。」and save it. Show the quiet completion feedback.

**Narration:**

> There is one large field and one action. imakore saves the thought and timestamp automatically, clears the field, and keeps unfinished drafts if I leave.

### 0:40–1:05 — Immediate demo experience

**Shot:** return to “振り返る.” If starting completely empty, use “サンプルで体験する.” Scroll to the insight card.

**Narration:**

> To make the idea testable immediately, a new user can load removable sample thoughts. Nothing is categorized by hand.

### 1:05–1:35 — Core insight loop

**Shot:** tap「写真」and show only related thoughts. Open one detail, go back, then tap「音楽」.

**Narration:**

> On the device, Japanese text is segmented and words that genuinely recur across different thoughts become quiet themes. Tapping one returns me to the moments behind it. The goal is reflection—not turning every thought into a task.

### 1:35–1:55 — Product completeness and privacy

**Shot:** search for「表現」, clear search, expand「記録を守る」and show export/import.

**Narration:**

> The MVP includes search, editing, deletion, offline use, dark mode, and a portable backup. Thought text and analysis stay in this browser; there is no account or external analytics.

### 1:55–2:25 — Codex and GPT-5.6

**Shot:** show the repository tree, tests passing, then the SwiftUI and PWA folders. Optionally show a short Codex session clip.

**Narration — verify the model wording before recording:**

> I built imakore with Codex using GPT-5.6 as my product and engineering partner. Codex helped turn a personal problem into a narrow specification, implement both a SwiftUI prototype and this production PWA, design the local analysis, improve accessibility, run build and regression checks, deploy the app, and prepare this submission. The key contribution was not adding more features—it was preserving the principle that users should never have to organize their own thoughts.

### 2:25–2:40 — Close

**Shot:** return to the insight card and the imakore name.

**Narration:**

> imakore means “right now, this.” Write what is here now, and later notice what keeps returning.

## Editing checklist

- Keep the final video below 3:00.
- Add English subtitles if narration is Japanese, or Japanese subtitles if narration is English.
- Avoid showing private thoughts or account details.
- Make the first visible insight appear before 1:10.
- End with the product name and live URL on screen.
